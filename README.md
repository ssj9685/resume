### 새로운 시도를 좋아합니다.

새로운 제품의 관리자 사이트를 간략히 만들 수 있는 기회가 왔을 때 기존에 존재하던 DB 데이터를 활용하여 풀스택으로 진행하였습니다.
그리고 기존의 Vue와 CoreUI 템플릿으로 구성되어있던 관리자 사이트를 React + Next로 전체 리뉴얼 하였습니다.

### 신기술에 관심이 많습니다.

최신 정보를 듣게 되면 그것을 팀원들과 공유하고 적극적으로 현재 적용해볼 수 있도록 합니다.
예를 들면 Fast API가 생산적이라는 얘기를 듣고 알고리즘 문제를 크롤링하여 슬렉으로 보내는 사이드 프로젝트를 진행하였습니다.
이러한 성격 덕분에 새로운 것에 대한 학습 속도가 향상되었다고 생각합니다. 

### 효율적인 업무 처리를 좋아합니다.

기존에 git-flow를 통해 협업을 진행하였는데, 업무의 흐름이 빠른 스타트업 특성상 딜레이가 걸리는 느낌을 받았습니다.
때문에 Trunked-based로 전환하여 업무 흐름을 개선하였습니다. 브렌치 하나를 잘 관리해야하기에 코드 리뷰에 더 신경쓰게 되었습니다.
팀 구성원들의 단순/루틴 업무 시간을 줄여주기 위해 지속해서 앱을 개발한 경험이 있습니다.

## 프로젝트

### 전기차 충전기 연동 앱 개발(EVpay) 및 운영
자사 전기차 충전기와 연동되는 앱을 개발하였습니다. 초기 디자인, 와이어프레임, 기술 스택 선정, 아키텍처 설계, 개발 및 배포까지 담당하였습니다. Flutter로 개발을 진행하였으며 결제 모듈의 AOS/iOS의 네이티브 앱과 Web의 호환을 위해 Flutter Javascript interop를 활용하였습니다. Github actions와 Fastlane을 활용하여 배포 자동화를 하였습니다. 앱의 보안을 위해 위변조를 감지하는 기능을 개발하였습니다. 이를 위하여 빌드 된 안드로이드 앱에서는 리버싱을 진행하여 smali코드를 변조, iOS에서는 flutter의 jailbreak 패키지를 활용하여 기능 테스트를 진행했습니다.

Google play: https://play.google.com/store/apps/details?id=kr.co.evar.evpay

App store: https://apps.apple.com/kr/app/evpay/id1619603697

### 전기차 충전기 관리자 사이트 리뉴얼 및 유지 보수
자사 전기차 충전기를 사용하는 충전 사업자들에게 충전기의 여러 정보를 보여주는 관리자 사이
트를 리뉴얼하였습니다. 기존 Vue.js에서 React.js + Next.js로 기술 스택을 변경하고 사업자
별로 여러개의 브랜치로 나뉘어서 배포되어 있던 것을 하나의 브랜치로 통합하였습니다. 잘 관리
된 브랜치를 기반으로 production/develop의 환경을 나누어 Github actions를 통해 CI/CD
파이프라인을 구축하였습니다. 일본과 미국에 있는 사업자들을 위해 i18n을 적용하여 현지화 작
업을 진행하였습니다.

### 전기차 충전기 설정 앱 개발
자사 전기차 충전기의 보드와 통신하여 사용하는 블루투스, 와이파이, NFC 모듈을 설정할 수 있
는 앱을 개발하였습니다. 기존에 수동으로 충전기의 설정을 변경하는 것이 어렵다는 펌웨어 팀의
고민을 해결하고자 여러 정보를 한번에 입력, 수정, 삭제할 수 있고 복사, 붙여넣기 기능이 있는
앱을 만들어 사내에 공유하였습니다.
